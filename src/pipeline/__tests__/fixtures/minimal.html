Functional programming emphasizes immutability, pure functions, and declarative code. Instead of mutating state and writing step-by-step instructions, functional programs describe transformations on data. Languages like Haskell, Clojure, and Elm embrace this paradigm fully, while languages like JavaScript, Python, and Rust incorporate functional features alongside other paradigms.

The benefits of functional programming include easier reasoning about code, better testability, and natural support for parallelism. Pure functions always produce the same output for the same input and have no side effects, making them predictable and easy to test in isolation.

Common functional patterns include map, filter, and reduce for transforming collections; higher-order functions that accept or return other functions; and pattern matching for handling different data shapes. These patterns lead to concise, expressive code that clearly communicates intent.
